Programming Comments:

/* */comment: can appear within a line or across lines. /* can be the first element on a line or can follow other elements; */ can be the final element on the line or can precede other elements.

//comment: says the remainder of the line is a comment; it can be the first element on the line or follow other elements.

;comment: specifies that the remainder of the line is a comment; it can be the first element on the line or can follow other elements.

;;comment: a weird case of the ; comment type, makes the comment available to the $TEXT function when the routine is distributed as object code only; the comment is only available to $TEXT if no commands precede it on the line.

Use SET to assign a value to a variable:

SET x = 100 command names and function names, are not case-sensitive variable names and method names, are case-sensitive

A statement cannot start on the first character position on a line; all commands must be indented. Comments must also be indented. The only code element that can appear in the first character position on a line is a label:

MyLabel SET x = 100 WRITE x

In this sample WRITE writes three arguments: the literal string “Hello ”, the “!” character (which is a symbolic operator specific to the WRITE command that issues a

line feed/carriage return) and the localvariable x, which is replaced during execution by its current value SET x =”World” WRITE “Hello”, ! , x
“$” prefix for function names. Function can be system or user defined (like all other langs, durr!! ).

Invalid variables name return <SYNTAX> errors Invalid variable name begins with a “_” character followed by a letter returns <

Order of Operations:

There is no “please excuse my dear aunt sally” order ( “()”, ^, *, /, +, -, ). There is only

parenthesizes, and left to right.

booleans work like math operators that return 1 or 0, like this

Set Value = (4 + 7 > 6 + 6) // 7 Value = (11 > 6 + 6) // 7 Value = (1 + 6) // 7 Write Value

Booleans logic works left to right to so add lots and lots of parenthesizes.

CommonCommands


$Piece

finds the spec’d item in a delimited string

Write $Piece(“ListString”,”QuotedDelimiter”,ItemNumber) ; example Write $Piece(“Kennedy Johnson Nixon”,” “,2) ; returns Johnson Write $Piece(“Nixon***Ford***Carter***Reagan”,”***”,

1,2) ; returns Nixon***Ford


$Data

Form: $D(variable) Retn: 0 – Variable is undefined 1 – Variable is defined but has no descendants 10 – Variable is undefined but has descendants 11 – Variable is defined and has descendants Std: Complies exactly Eg: K A S A(1)=42 W $D(A) -> 10


$Order

Returns next subscript at the same level of a given variable. Also loops through a list of local variables set in a partition.

%=1, %USER=”mikel”, var=123.

Write $Order(@(“”)) ; this returns “%”

Write $Order(%) ; this returns “%USER”

Write $Order(%USER) ; this returns “var”


$Justify




Retn: expr space padded on the left to a length of int1 characters.
If int2 is specified, expr is first rounded to int2 decimal places.


Eg: $J(“ABC”,5) -> ” ABC”
$J(.456,6,2) -> ” 0.46″

Namespaces and Databases

Data and code is stored in databases referred to by namespaces. namespaces = logical reference to several storage location

Changing Namespace from Terminal: ‘

do ^%CD’or ‘zn “Namespace”‘ Each database has a cache.dat in the OS

Databases contain Globals and Routines
